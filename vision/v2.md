# Version 2: Connected Personal Memory

## Purpose

Version 2 should begin only after Trusted External Memory succeeds in real use.

V2 is not the full Chief of Staff product. It is the next layer after V1 proves that Ty can capture messy context and reliably recover it later with source proof.

The goal of V2 is connected personal memory:

- Memories start linking to each other.
- People, projects, ideas, commitments, preferences, and recurring topics become easier to revisit over time.
- Corrections teach the system what Ty means.
- Recall becomes more contextual without becoming noisy.

Do not implement V2 concepts in the active codebase until real-world V1 usage shows where the current memory model breaks.

## Product Thesis

V1 answers:

Can Ty trust the system to preserve and recover messy context?

V2 answers:

Can the system connect that context across time without making Ty maintain structure manually?

The product is still for Ty first. The question is not whether this is the smallest startup product. The question is whether the system is becoming the personal memory and organization layer Ty wishes existed.

## What V2 Should Improve

- Entity resolution: understand that "Sarah", "Sarah Chen", and "pricing Sarah" may refer to the same person.
- Connected recall: answer "what do I know about this person/project/idea?" with source-backed memory.
- Memory consolidation: merge repeated notes into a clearer durable memory without losing raw sources.
- Correction learning: remember when Ty edits, rejects, promotes, or adds context.
- Recurring themes: detect ideas, interests, projects, and obligations that appear repeatedly.
- Preference capture: begin preserving durable Ty-specific preferences and constraints.
- Context pages: lightweight views for important people, projects, ideas, and recurring topics.
- Better confidence: distinguish direct facts, inferred connections, stale memory, and user-confirmed memory.

## What V2 Should Not Become

V2 should not become:

- A CRM
- A task manager
- A calendar app
- A travel planner
- A gift reminder app
- A skills tracker
- An SBIR tracker
- A dashboard product
- A proactive research agent

Those domains may eventually sit on top of the memory layer, but V2 should still strengthen memory itself.

## Likely V2 Capabilities

### Entity Pages

Create simple source-backed pages for recurring entities such as:

- People
- Projects
- Ideas
- Companies
- Places
- Topics

These pages should summarize what the system knows, show supporting sources, and expose uncertainty.

### Relationship Awareness

Begin connecting memory objects:

- Person related to project
- Person mentioned at event
- Idea related to project
- Commitment owed to person
- Preference related to travel, tools, work style, family, or business
- Opportunity related to person, company, or idea

Relationship creation should remain conservative and explainable.

### Memory Consolidation

Repeated captures should be allowed to form a stronger memory.

Example:

- Raw note 1: "Mike insurance guy mentioned renewals."
- Raw note 2: "Carla also said agencies miss renewal follow-up."
- Raw note 3: "Insurance AI idea keeps coming back: renewal follow-through."

V2 should eventually recognize this as a recurring opportunity theme while preserving every source.

### Correction Learning

Corrections should become training signals:

- Not relevant
- Same person as another memory
- Different person with similar name
- Promote raw source to memory
- Add missing context
- Dismiss extraction
- Merge duplicates
- Mark stale

The system should learn what Ty considers useful, not merely store final edited text.

## V2 Success Criteria

V2 succeeds if:

- Ty can ask about a person, project, idea, or topic and get a coherent source-backed answer.
- The system connects repeated thoughts without requiring manual folders or tags.
- Corrections make future recall better.
- The memory layer becomes more useful with each week of capture.
- Ty trusts the system more because uncertainty is visible.
- New structure reduces cognitive load instead of creating review homework.

## Entry Criteria

Do not start V2 until V1 shows:

- Daily capture is fast enough on laptop and phone.
- Raw source preservation is reliable.
- Recall is useful across hundreds or thousands of captures.
- False positives are controlled enough that Ty does not lose trust.
- The current flat memory model produces repeated real-world pain.

## Architectural Constraint

V2 should introduce structure only where V1 usage proves the need.

Future concepts such as entities, relationships, observations, preferences, corrections, provenance, and salience should remain optional architecture until they solve observed failures.
